# A 股 K 线数据源选型与缓存增量方案

## 一、你的需求

1. **全 A 股 K 线拉取**并**本地缓存**
2. **次日只拉取最新 K 线**（增量更新）
3. **对全市场按 K 线计算打分**，按你的限制（如 10 条）返回**评分最高的 N 条**
4. **今日预测**同理：全市场打分后取**看涨概率最高的 N 条**

## 二、数据源选型结论

| 数据源 | 是否最适合 | 说明 |
|--------|-------------|------|
| **pytdx** | ✅ **当前优先** | 免费、通达信协议，可做全市场日线；已实现自建增量逻辑与缓存（按股票拉取最近 800 条，只写入比缓存新的交易日）。无需 token，安装即用。 |
| Tushare Pro | 后续可切换 | 按 `trade_date` 一次可拉全市场当日日线，适合「按日增量」；需付费积分。设置 `PREFER_TUSHARE=1` 且配置 `TUSHARE_TOKEN` 后优先走 Tushare。 |
| AllTick | 免费备选 | 免费、多市场，可评估 A 股日线接口与限流。 |
| AKShare | 兜底/选股实时 | 免费但爬虫式、易慢/限流；选股/预测在无缓存或单只补数时仍会用到。 |
| iTick | 不优先 | 更偏实时/Level-2，日线回测与全量缓存非其主场景。 |

**结论：当前实现优先使用 pytdx（免费）；等 Tushare 稳定或需要按日全市场一次拉取时，可改为优先 Tushare。**

## 三、整体架构（缓存 + 增量）

```
┌─────────────────────────────────────────────────────────────────┐
│  选股 / 今日预测（全量打分取 top N）                                │
│  1) 从缓存读取全 A 股日线（或按需先跑一次增量）                        │
│  2) 对每只计算得分/看涨概率                                          │
│  3) 排序后取前 N 条                                                  │
└─────────────────────────────────────────────────────────────────┘
                                    ▲
                                    │ 读
┌─────────────────────────────────────────────────────────────────┐
│  本地缓存（SQLite：code + trade_date + open/high/low/close/vol）   │
│  - 首次：全量写入（或按 trade_date 批量拉 Tushare）                  │
│  - 之后：只拉「最新交易日」数据并 append（增量）                      │
└─────────────────────────────────────────────────────────────────┘
                                    ▲
                                    │ 写（仅增量或补全）
┌─────────────────────────────────────────────────────────────────┐
│  数据源（可插拔）                                                   │
│  - pytdx（优先）：按股票 get_security_bars(9, market, code, 0, 800) │
│    只写入 trade_date > 缓存最大日期的行 → 免费、增量                 │
│  - Tushare Pro：pro.daily(trade_date=日期) 按日全市场 → 设置 PREFER  │
│  - AKShare：选股/预测单只补数或未装 pytdx 时的兜底                   │
└─────────────────────────────────────────────────────────────────┘
```

- **今日/盘后**：先跑一次「增量」（只拉取最新一个或几个交易日），写入缓存。
- **选股 / 今日预测**：只读缓存中的全 A 股日线，在内存里算分并取 top N，不再实时逐只请求接口，因此会快很多。

## 四、pytdx 增量方式（当前默认）

- 安装：`pip install pytdx`。
- 增量流程（`ensure_incremental()`）：
  1. 连接通达信行情服务器（内置多主机可轮询）；
  2. 获取沪/深 A 股列表（`get_security_list(0,0)` / `(1,0)`），过滤 6 位数字代码；
  3. 对每只股票请求日 K：`get_security_bars(9, market, code, 0, 800)`；
  4. 只保留 `trade_date > 缓存最大日期` 的行写入 SQLite；
  5. 首次运行无缓存时，会写入每只股票最近 800 条日线。
- **次日再跑**：只拉取比缓存新的交易日，实现「只拉最新 K 线」。

## 五、Tushare Pro 增量方式（可选）

- 设置 `PREFER_TUSHARE=1` 且配置 `TUSHARE_TOKEN` 后，将改用 Tushare。
- 日线接口：`pro.daily(trade_date='YYYYMMDD')` 按日全市场拉取，增量更快（按日而非按股）。

## 六、实现状态

- 见 `core/stock_kline_cache.py`：可插拔数据层 + SQLite 缓存；**默认优先 pytdx**，失败或未安装时再试 Tushare；未配置 Tushare 时选股/预测仍可用 AKShare 实时拉取（或从缓存读）。
